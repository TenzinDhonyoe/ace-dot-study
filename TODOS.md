# TODOS — ace-dot-study

Deferred scope for Phase 2.5+. Each item has a trigger that decides when it
promotes out of this file.

## Deferred from Phase 2 planning

### Incremental "add lecture" regeneration
Let users drop additional PDFs mid-semester and regen only the affected
sections instead of the whole site. Needs delta-aware generator prompt
changes. **Trigger:** 3+ users ask for it, or a complaint about "I added a
lecture and lost my flashcard progress" shows up.

### PIN-based cross-device localStorage sync
4-digit PIN binds a generated site's widget state (Got-it, MCQ answers)
across devices via Upstash. **Trigger:** analytics shows >30% of sites get
opened on both desktop and mobile.

### Course-code namespace (`ace.study/BME804`) + upvotes
Wikipedia-for-finals vision: a canonical community site per course code.
Requires real curation + moderation + anti-spam. **Trigger:** organic
remix chains reach depth 5+ (signal that a community is forming).

### Auto-parse course code from PDF headers
Preset `meta.course` and `meta.title` from detected course code in PDF
text. Zero typing. **Trigger:** user testing shows typing fatigue on the
upload page.

### Remix watermark trail
Display `generated by anon · remixed by anon · remixed by you` on every
site. Quiet social signal. **Trigger:** first 100 remixes land.

### Professor-branded tier + custom domains
`BME804.yourprof.edu` with institutional branding. **Trigger:** 3+
professor requests land in GitHub issues or email.

### Accounts + billing (Phase 2.5)
Introduce accounts as *labels* on existing slugs, never as gates.
Stripe + magic-link. **Trigger:** 1000 weekly active generated sites.

### Creator analytics dashboard
"Your site was viewed 42 times, 8 people remixed it." Requires logged-in
creator identity → blocked on the accounts TODO. **Trigger:** same.

### Global spend-cap alerting
Slack webhook at 50% and 80% of `MAX_DAILY_ANTHROPIC_USD`. Currently we
only surface the cap to end users when it's hit. **Trigger:** first time
the cap actually fires in production.

### Sentry for client-side errors
Vercel's built-in error reporting is server-side. Client errors (pdf.js
crashes, SSE reconnect bugs) go uncaught. **Trigger:** first user report
of "the spinner spins forever."

## Deferred from eng review

### KV → Durable Object migration for spend counter
Currently using Upstash `INCR` (atomic, simple). If we ever see Upstash
rate-limit pressure at scale, migrate the counter to a Cloudflare Durable
Object or Vercel Durable Storage. **Trigger:** 10k generations/day.

### Move `eval/` from the OSS monorepo into its own npm package
Right now it's a git submodule or copy-in. A dedicated `ace-study-eval`
package makes it reusable by third-party generators. **Trigger:** someone
outside the core team publishes a competing generator.

### Cache widget bundle in Blob, stop relying on unpkg
`unpkg.com` is a single point of failure for the widget bundle at page
load time. Copy to Blob at deploy, rewrite URLs. **Trigger:** first unpkg
outage that breaks production.
