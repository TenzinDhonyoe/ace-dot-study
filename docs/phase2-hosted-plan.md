# CEO Plan: Ace Phase 2 — Hosted Web App (`ace.study`)

Generated by `/plan-ceo-review` on 2026-04-21
Mode: SCOPE EXPANSION
Approach: B — Edge-first, zero-backend
Target repo: NEW (separate from the OSS ace monorepo)

## Vision

### 10x Reframe

`ace.study` is not "a hosted generator." It is the **anonymous, shareable link layer for AI-generated interactive study content.** Every generated site is a vanity URL a student drops in their group chat at 4pm the day before finals. Remix turns every consumer into a creator. No accounts. No login walls. Links are the product.

This sits downstream of the OSS library (`ace-study-components`, `ace-study-template`), not above it. The library stays the real product. The hosted app is a thin edge-runtime skin that reaches the students who can't `git clone`.

### Platonic Ideal (12 months)

McGill biomed student finishes her last lecture Thursday 4pm. Opens `ace.study` on her phone. Drags three PDFs from Files. Taps Generate. 90 seconds of live-narrated progress. Gets a link. Drops it in Discord. Six classmates open it — sliders work, flashcards work, progress persists per-device. One hits Remix, adds his handwritten notes, regenerates the microactuators section. She studies from his version on the bus Friday. She aces the exam. Nobody made an account. Nobody paid.

Monetization is Phase 2.5 — professor-branded canonical course sites. Not now.

### North Star

The shareable link layer for AI-generated interactives. Virality via remix trail, not via login wall.

## Accepted v1 Scope

1. **Cloudflare Workers + R2 + KV** edge runtime. No VMs, no DB.
2. **Astro** frontend, mobile-first, static-first with islands for interactive bits.
3. **Client-side PDF parsing** via `pdf.js`. Vision-API fallback for scanned PDFs.
4. **Streaming generation** with live model-narration events (SSE).
5. **Anonymous shareable vanity URLs** (`ace.study/<8-char-nanoid>`).
6. **Remix** — every site exposes Remix button; clones `config.json`, reruns generation with user overrides/additional PDFs.
7. **Exam-date countdown banner** when user provides an exam date.
8. **Discord/Reddit OG share cards** for link previews.
9. **Shared house Anthropic key**, aggressively rate-limited (3 generations/day/IP). **BYOK escape hatch** for power users.
10. **Cloudflare Turnstile** (invisible CAPTCHA) + per-IP KV rate-limit + **hard global daily spend cap** (kill switch).
11. **OSS prerequisite:** refactor `ace-study-template` to expose a pure-function `renderSite(config) → htmlString` API (v0.3).

## NOT in scope (v1)

- Accounts, login, password reset, email magic links
- Payments, subscriptions, any billing
- Professor-branded / "official" course sites
- Incremental mid-semester regen
- PIN-based cross-device sync
- Course-code namespace + community curation
- Auto-parse course code from PDFs
- Remix watermark / chain display
- Mobile native apps
- Custom domain mapping
- Creator-facing analytics dashboards
- Any feature that requires a relational DB

  │  - Turnstile verify                                              │
  │  - Global spend kill-switch (abort if $MAX_DAILY_SPEND hit)      │
  │  - Stream Anthropic API with ace-review system prompt            │

  │  - Retry malformed chunks 2x                                     │

  │  R2 (public read): /{slug}/index.html, /{slug}/config.json       │
  │  Worker serves static HTML at ace.study/{slug}                   │
  │  Remix flow: GET config.json → edit → POST /api/generate         │

## Error & Rescue Registry

| Codepath | Failure | Rescued? | Action | User sees |
|---|---|---|---|---|
| `pdf.js` extract | No text layer (scanned PDF) | Y | Fallback to Claude Vision | "Scanning image PDF…" |
| `pdf.js` extract | Password-protected | Y | Abort | "PDF is encrypted — remove password and retry" |
| `/api/generate` | Anthropic 429 | Y | Backoff + retry 2x, then degrade | "High load — retrying" |
| `/api/generate` | Anthropic 5xx | Y | Retry 2x, then partial-site fallback | "Section N failed — retry or skip" |
| `/api/generate` | Malformed JSON from model | Y | Retry with stricter prompt, then skip chunk | Inline banner |
| `/api/generate` | Prompt injection in PDF text | Partial | Strip obvious patterns; system-prompt hardening | Transparent |
| R2 write | Quota exceeded | **N — GAP, resolve in eng review** | — | 500 |
| KV read | Timeout | **N — GAP, resolve in eng review** | — | 500 |
| Global spend cap hit | Anthropic calls blocked | Y | Kill switch; queue off | "Daily free quota reached — try BYOK or tomorrow" |

## Security — Top Findings

- **🔴 CRITICAL:** Hard-coded global daily spend cap. Without it, one attacker = 10x bill in an hour.
- **⚠️** Prompt-injection hardening: system prompt must defend against PDF-embedded instructions.
- **⚠️** R2 public read = generated content is world-visible. Upload page must warn: "Don't upload docs with PII."
- **⚠️** Rate-limit layers: per-IP (KV) + Turnstile + global cap. All three needed.

## Success Criteria (hosted app)

- **2 weeks post-launch:** 50 generated sites, 10 remixes, 3 sites shared in external Discord/Reddit.
- **3 months:** 500 generated sites, at least one organically-shared site hits 100+ viewers.
- **Triggers Phase 2.5:** 1000 weekly active sites OR 3 professor requests for custom branding.

## Failure Modes

- **Generator quality collapses at scale** — mitigated by eval harness reuse, spot-check loop, partial-site fallback.
- **Cost blowout** — hard global daily cap is the last line of defense. Rate-limit + Turnstile are the first two.
- **Drift between OSS renderer and hosted renderer** — mitigated by refactoring OSS first (pure-function API) and importing, not forking.
- **No viral uptake** — if remix+share doesn't happen within 30 days of launch, revisit the wedge. Do not add more features on a dead trunk.

## Design Plan (from `/plan-design-review`, 2026-04-21)

### Visual DNA — inherit the OSS palette

The hosted app must feel continuous with the generated sites. Students who land
on `ace.study` and end up on their generated site should experience one surface,
not two.

- Palette: `--paper #faf8f3`, `--ink #1a1a1a`, `--muted #6b6257`, `--rule #e8e2d6`,
  `--accent #b8451f` (rust), `--mark #fff3c4` (highlighter).
- Typography: Charter / Iowan Old Style serif for body + headings; system-ui for
  controls and small UI text.
- Vibe: editorial, textbook, reading room. Explicit rejection of: SaaS blue,
  drop-shadow cards, gradient CTAs, stock illustrations.
- Reuse the OSS CSS variable contract (`--ace-paper`, etc.) so the landing page
  and the generated site share one source of truth.

### AI-slop avoidance — explicit anti-patterns

- No "Powered by AI ✨" copy anywhere.
- No three-feature-cards grid.
- No gradient hero headline.
- No stock illustration of a student with a laptop.
- No emoji-heavy empty states.
- Dark mode is a separate palette, not `invert(1)`.

### SSE event schema (contract)

```ts
// packages/shared-types/src/sse-events.ts
export type GenerateEvent =
  | { type: 'slug';             slug: string }                      // FIRST event
  | { type: 'narration';        text: string }
  | { type: 'section_start';    id: string; title: string }
  | { type: 'section_complete'; id: string }
  | { type: 'section_failed';   id: string; reason: string }
  | { type: 'validation_error'; chunk: string; retry: number }
  | { type: 'complete';         slug: string; partial: boolean }
  | { type: 'error';            code: ErrorCode; message: string; retryable: boolean };

export type ErrorCode =
  | 'rate_limited' | 'spend_cap_hit' | 'turnstile_failed'
  | 'anthropic_down' | 'invalid_pdf' | 'internal';
```

### Request flow for `/api/generate`

```
  1. Parse Turnstile token from header              fast, free
  2. Check DO spend counter                         ~10ms, atomic
  3. Check per-IP KV rate-limit                     ~5ms
  4. Verify Turnstile token server-side             ~50ms, spends Turnstile quota
  5. Increment per-IP KV counter                    ~5ms write
  6. Emit SSE { type: 'slug', slug }                immediate
  7. Stream Anthropic API (60-120s)                 EXPENSIVE
       ├── on each section: validate against schema
       ├── on malformed JSON: retry 2x, then skip + section_failed event
       └── on 429/5xx: backoff + retry 2x, then degrade
  8. DO increment global spend counter              atomic
  9. Write config.json + index.html to R2           ~40ms
 10. Emit SSE { type: 'complete', partial? }        close stream
```
